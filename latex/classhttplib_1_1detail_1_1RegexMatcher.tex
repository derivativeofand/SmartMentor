\doxysubsection{Detailed Description}
Performs std\+::regex\+\_\+match on request path and stores the result in Request\+::matches

Note that regex match is performed directly on the whole request. This means that wildcard patterns may match multiple path segments with /\+: "{}/begin/(.\texorpdfstring{$\ast$}{*})/end"{} will match both "{}/begin/middle/end"{} and "{}/begin/1/2/end"{}.
